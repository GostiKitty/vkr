## Geometry — Геометрия

### G1. Площадь произвольного многоугольника

`geom_polygon_area`

$$
A = \frac{1}{2}\left|\sum_{i=1}^{n} x_i y_{i+1} - x_{i+1} y_i\right|
$$

Формула шнуровки используется для расчета площади контура помещения на плане.

**Физический смысл.** Дает реальную геометрическую площадь помещения по координатам вершин.

**Результат.** Результат используется для объема, удельных нагрузок и инженерных расчетов.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $A$ | площадь полигона | м² | выход |
| $x_i, y_i$ | координаты вершин | м | геометрия модели |
| $n$ | число вершин | — | геометрия модели |

**Метод:** Shoelace formula

**Применимость:** Плоские замкнутые полигоны без самопересечений.

**Допущения:** Контур замкнут; Вершины перечислены по порядку обхода


### G2. Объем помещения

`geom_volume`

$$
V = A \cdot h
$$

Объем помещения принимается как площадь пола, умноженная на высоту уровня.

**Физический смысл.** Дает геометрический объем воздуха в помещении.

**Результат.** Используется для воздухообмена, теплоемкости воздуха и баланса теплоты.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $V$ | объем помещения | м³ | выход |
| $A$ | площадь пола | м² | геометрия модели |
| $h$ | высота помещения или уровня | м | геометрия модели |

**Метод:** Призматическая геометрическая модель

**Применимость:** Помещения с постоянной высотой по плану.

**Допущения:** Высота помещения принимается равной высоте уровня или заданной высоте комнаты


### G3. Отапливаемый объём здания

*ядро расчёта · Геометрия · влияет на солвер*  `geometry_heated_volume`

$$
V_{ot} = \sum_i A_i \cdot h_i
$$

Суммарный отапливаемый объём по комнатам модели: площадь пола на высоту уровня.

**Физический смысл.** Определяет объём воздуха в отапливаемой части здания.

**Результат.** Используется в n = V̇·3600/V и в q_от, Q_год.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $V_{ot}$ | отапливаемый объём | м³ | выход |
| $A_i$ | площадь помещения | м² | геометрия |
| $h_i$ | высота | м | уровень |

**Метод:** Геометрическая сумма по помещениям

**Применимость:** Инфильтрация, ACH, нормативные показатели СП 50, согласованность расчёта.

**Допущения:** A_i — площадь контура комнаты; h_i — высота уровня или 3 м по умолчанию

**Код:** `src/core/thermal/infiltration.ts`, `src/core/thermal/consistencyCheck.ts`


### G4. Площадь проёма

*ядро расчёта · Геометрия · влияет на солвер*  `geometry_opening_area`

$$
A_{opening} = b \cdot h
$$

Площадь окна или двери по габаритам проёма в модели.

**Физический смысл.** Связывает размеры проёма с теплопотерями и солнечными притоками.

**Результат.** Суммируется в площади остекления и дверей здания.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $A_{opening}$ | площадь проёма | м² | выход |
| $b$ | ширина | м | модель |
| $h$ | высота | м | модель |

**Метод:** Прямоугольный проём

**Применимость:** Площадно-взвешенные U, WWR, солнечные и инфильтрационные расчёты.

**Допущения:** Проём прямоугольный; Ширина и высота заданы в метрах

**Код:** `src/shared/utils/openingThermalData.ts`


## Envelope — Ограждающие конструкции

### E1. Сопротивление теплопередаче слоя

`layer_resistance`

$$
R = \frac{d}{\lambda}
$$

Термическое сопротивление однородного слоя рассчитывается по толщине и теплопроводности материала.

**Физический смысл.** Показывает, насколько слой препятствует тепловому потоку.

**Результат.** Чем больше R, тем меньше теплопотери через слой при прочих равных.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $R$ | термическое сопротивление слоя | м²·К/Вт | выход |
| $d$ | толщина слоя | м | материал/конструкция |
| $\lambda$ | коэффициент теплопроводности | Вт/(м·К) | материал |

**Метод:** Стационарная одномерная теплопроводность

**Применимость:** Однородные слои в плоской ограждающей конструкции.

**Допущения:** Материал слоя однороден; Теплопередача рассматривается по толщине слоя


### E2. Суммарное сопротивление и U-значение конструкции

`total_resistance`

$$
R_{total} = R_{si} + \sum_i \frac{d_i}{\lambda_i} + R_{se}, \quad U = \frac{1}{R_{total}}
$$

Сопротивления слоев и поверхностные сопротивления складываются, после чего находится U-значение ограждения.

**Физический смысл.** Связывает состав конструкции с общей теплопередачей через ограждение.

**Результат.** U используется далее в формуле потерь через ограждение.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $R_{total}$ | суммарное сопротивление теплопередаче | м²·К/Вт | расчет |
| $R_{si}$ | внутреннее поверхностное сопротивление | м²·К/Вт | допущение/норма |
| $R_{se}$ | наружное поверхностное сопротивление | м²·К/Вт | допущение/норма |
| $U$ | коэффициент теплопередачи | Вт/(м²·К) | расчет |

**Метод:** Расчет многослойной ограждающей конструкции

**Применимость:** Многослойные стены, покрытия, перекрытия при стационарном режиме.

**Допущения:** Слои соединены последовательно; Используются эквивалентные поверхностные сопротивления


### E3. Теплопотери через ограждение

`envelope_heat_loss`

$$
\dot{Q} = U \cdot A \cdot (T_{in} - T_{out})
$$

## Thermal — Теплофизика и расчётные контуры

### T1. Тепловой баланс помещения

`thermal_balance`

$$
Q_{balance} = Q_{gains} - Q_{losses}, \quad Q_{deficit} = \max(Q_{losses} - Q_{gains}, 0)
$$

Итоговый стационарный баланс притоков и потерь в помещении.

**Физический смысл.** Показывает, хватает ли имеющихся теплопритоков для компенсации теплопотерь.

**Результат.** Q_deficit равен требуемой дополнительной мощности отопления.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $Q_{balance}$ | итоговый тепловой баланс | Вт | расчет |
| $Q_{gains}$ | сумма теплопритоков | Вт | расчет |
| $Q_{losses}$ | сумма теплопотерь | Вт | расчет |
| $Q_{deficit}$ | требуемая мощность отопления | Вт | выход |

**Метод:** Стационарный тепловой баланс

**Применимость:** Инженерная оценка требуемой мощности отопления на расчетный момент.

**Допущения:** Притоки и потери суммируются по мощности; Баланс относится к выбранному расчетному срезу


### T2. Тепловой баланс помещения

`thermal_balance_room`

$$
Q_{balance} = Q_{gains} - Q_{losses}
$$

Короткая версия баланса для подсказок в редакторе и свойствах помещений.

**Физический смысл.** Показывает знак избытка или дефицита теплоты в помещении.

**Результат.** Положительный баланс означает избыток, отрицательный - дефицит теплоты.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $Q_{balance}$ | итоговый тепловой баланс | Вт | расчет |
| $Q_{gains}$ | теплопритоки | Вт | расчет |

**Допущения:** Используется тот же баланс, что и в инженерной вкладке результатов


### T3. Сосредоточенная RC-модель помещения

`rc_lumped`

$$
C_{eff}\frac{dT}{dt} = Q_{in} - Q_{out}, \quad T_{k+1} = T_k + \frac{\Delta t}{C_{eff}}(Q_{in} - Q_{out})
$$

Модель первого порядка используется для прогноза температуры и сценарного сравнения.

**Физический смысл.** Определяет изменение средней температуры по балансу мощностей и эффективной теплоемкости.

**Результат.** Результат показывает, как будет меняться средняя температура помещения шаг за шагом.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $C_{eff}$ | эффективная теплоемкость | Дж/К | расчет |
| $T_k$ | температура на текущем шаге | °C | состояние модели |
| $T_{k+1}$ | температура на следующем шаге | °C | выход |
| $\Delta t$ | шаг по времени | с | настройки расчета |
| $Q_{in}$ | теплопритоки | Вт | расчет |
| $Q_{out}$ | теплопотери | Вт | расчет |

**Метод:** Lumped capacitance / RC model

**Применимость:** Прогноз средней температуры помещения во времени.

**Допущения:** Помещение описывается одной усредненной температурой; Температурное поле по объему не разрешается


### T4. Стационарное температурное поле на сетке

`steady_field`

$$
\nabla \cdot (k_{eff}\nabla T) + q = 0, \quad T_{i,j} = \frac{T_E + T_W + T_N + T_S + q_{i,j}h^2/k_{eff}}{4}
$$

Температурная карта строится конечно-разностным методом на равномерной 2D-сетке.

**Физический смысл.** Оценивает распределение температуры по плану с учетом источников и граничных условий.

**Результат.** Результат показывает локальные холодные и теплые зоны на сетке.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $k_{eff}$ | эффективная теплопроводность | Вт/(м·К) | модель поля |
| $q$ | объемный источник теплоты | Вт/м³ | источники тепла |
| $h$ | шаг сетки | м | настройки расчета |
| $T$ | температура | °C | решатель |

**Метод:** Пятиточечная конечно-разностная схема

**Применимость:** Плановая стационарная карта температуры помещения.

**Допущения:** Рассматривается 2D-проекция; Используется эффективная плановая теплопроводность/диффузия


### T5. Граничные условия теплопроводности

`boundary_conditions`

$$
T = T_{boundary}; \quad -k\frac{\partial T}{\partial n} = q_n; \quad -k\frac{\partial T}{\partial n} = h(T_{surface} - T_{air})
$$

Для расчета поля допускаются условия Дирихле, Неймана и Робина; в текущей карте используются фиксированные температуры поверхностей.

**Физический смысл.** Определяет, как расчетная область обменивается теплом с границами и средой.

**Результат.** От выбора граничных условий зависит форма температурного поля и уровень локальных экстремумов.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $T_{boundary}$ | заданная температура границы | °C | граничные условия |
| $q_n$ | нормальный тепловой поток | Вт/м² | граничные условия |
| $h$ | коэффициент теплоотдачи | Вт/(м²·К) | граничные условия |

**Метод:** Граничные условия Дирихле, Неймана и Робина

**Применимость:** Постановка стационарной задачи теплопроводности.

**Допущения:** В текущем интерфейсе карта использует упрощенное фиксирование температур внутренних поверхностей

### T7. Расход теплоносителя по тепловой нагрузке

## Uncertainty — Неопределённость и чувствительность

### U1. Коэффициент чувствительности

`sensitivity_index`

$$
S_x = \frac{\Delta Y / Y}{\Delta X / X}
$$

Локальная чувствительность оценивается конечным возмущением параметра и полным пересчетом результата.

**Физический смысл.** Показывает, насколько относительное изменение параметра влияет на относительное изменение результата.

**Результат.** По знаку видно направление влияния, по модулю - его силу.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $S_x$ | индекс чувствительности | безразм. | выход |
| $X$ | входной параметр | — | вход |
| $Y$ | выходная величина | — | выход |
| $\Delta X$ | изменение входного параметра | — | вариант |
| $\Delta Y$ | изменение выходной величины | — | пересчет |

**Метод:** Локальная чувствительность конечным возмущением

**Применимость:** Сравнение влияния U, ACH, температурного напора и других входов на требуемую мощность.

**Допущения:** Изменение параметра мало относительно базовой точки расчета


### U2. Среднее значение по Монте-Карло

`uncertainty_mc`

$$
\hat{\mu} = \frac{1}{N}\sum_{k=1}^{N} y_k
$$

Среднее по множеству прогонов при случайных входных параметрах.

**Физический смысл.** Показывает ожидаемое значение выходной метрики с учетом разброса входов.

## Calibration — Калибровка цифрового двойника

### C1. RMSE по месячным данным

`calibration_rmse`

$$
RMSE = \sqrt{\frac{1}{M}\sum_{m=1}^{M}(E_m^{obs} - E_m^{sim})^2}
$$

Классическая метрика расхождения между наблюдаемыми и расчетными месячными энергозатратами.

**Физический смысл.** Показывает средний масштаб ошибки модели в тех же единицах, что и энергия.

**Результат.** Меньший RMSE означает лучшее совпадение модели с наблюдениями.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $M$ | число наблюдений | — | данные калибровки |
| $E_m^{obs}$ | наблюдаемая энергия | кВт·ч | измерения |
| $E_m^{sim}$ | расчетная энергия | кВт·ч | модель |

**Метод:** Root Mean Square Error

**Применимость:** Калибровка модели по 12 месячным значениям или другой серии наблюдений.

**Допущения:** Все наблюдения даны в одинаковых единицах; Каждая точка имеет одинаковый вес, если не задано иное


### C2. MAPE по месячным данным

`calibration_mape`

$$
MAPE = \frac{100\%}{M}\sum_{m=1}^{M}\left|\frac{E_m^{obs} - E_m^{sim}}{E_m^{obs}}\right|
$$

Средняя относительная ошибка между расчетом и наблюдениями в процентах.

**Физический смысл.** Показывает типичную относительную ошибку модели.

**Результат.** Чем меньше MAPE, тем ближе модель к фактическим месячным данным.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $M$ | число наблюдений | — | данные калибровки |
| $E_m^{obs}$ | наблюдаемая энергия | кВт·ч | измерения |
| $E_m^{sim}$ | расчетная энергия | кВт·ч | модель |

**Метод:** Mean Absolute Percentage Error

**Применимость:** Сравнение качества калибровки в относительных единицах.

**Допущения:** Наблюдаемые значения не равны нулю; Все точки имеют одинаковый вес, если не задано иное


### C3. MBE валидации цифрового двойника

*производная / диагностика · Derived / diagnostics*  `derived_validation_mbe`

$$
MBE = \frac{\sum (M_i - S_i)}{\sum M_i} \cdot 100\%
$$

Mean Bias Error между измерениями и моделью.

**Физический смысл.** Систематическое смещение модели, %.

**Результат.** Derived-only валидация.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $M_i$ | измеренное значение | — | — |
| $S_i$ | смоделированное значение | — | — |

**Метод:** Twin validation

**Применимость:** Синхронизированные measured/simulated series.

**Допущения:** Не генерируются синтетические measured data

**Код:** `src/core/thermal/derived/buildingPerformanceMetrics.ts`


### C4. CVRMSE валидации цифрового двойника

*производная / диагностика · Derived / diagnostics*  `derived_validation_cvrmse`

$$
CVRMSE = \frac{\sqrt{\sum (M_i - S_i)^2 / (n-1)}}{mean(M)} \cdot 100\%
$$

Коэффициент вариации RMSE.

**Физический смысл.** Относительная ошибка модели, %.

**Результат.** Derived-only.

**Метод:** Twin validation

**Применимость:** n ≥ 2 синхронизированных точек.

**Допущения:** mean(M) ≠ 0

**Код:** `src/core/thermal/derived/buildingPerformanceMetrics.ts`


### C5. RMSE_T валидации цифрового двойника

*производная / диагностика · Derived / diagnostics*  `derived_validation_rmse`

$$
RMSE_T = \sqrt{\frac{\sum (T_{meas,i} - T_{sim,i})^2}{N}}
$$

Среднеквадратичная ошибка температуры.

**Физический смысл.** RMSE, °C.

**Результат.** Derived-only.

**Метод:** Twin validation

**Применимость:** Синхронизированные температурные ряды.

**Допущения:** Безопасное сопоставление по timestamp при разной длине

**Код:** `src/core/thermal/derived/buildingPerformanceMetrics.ts`


### C6. Стохастический подбор параметров

*legacy · Legacy*  `calibration_parameter_search`

$$
\theta^* = \arg\min_{\theta \in \Theta} J(\theta), \quad J \in \{RMSE, MAPE\}
$$

Случайный поиск по диапазонам параметров модели для минимизации расхождения с наблюдениями.

**Физический смысл.** Находит набор параметров, при котором модель ближе к измерениям.

**Результат.** Возвращает bestParameters, rmse и mape.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $\theta$ | вектор параметров | — | выход |
| $J$ | целевая функция | — | расчёт |

**Метод:** Random search calibration

**Применимость:** Legacy-калибровка и подгонка коэффициентов модели.

**Допущения:** Фиксированное число итераций; Модель детерминирована при заданном θ

**Код:** `src/core/calibration/calibrator.ts`


### C7. RMSE калибровки по ряду

*legacy · Legacy*  `calibration_timeseries_rmse`

$$
RMSE = \sqrt{\frac{1}{N}\sum_{i=1}^{N}(y_i^{obs} - y_i^{sim})^2}
$$

Среднеквадратичная ошибка между наблюдениями и предсказанием модели на каждом шаге ряда.

**Физический смысл.** Масштаб средней ошибки в единицах наблюдаемой величины.

**Результат.** Минимизируется при подборе параметров.

| Обозн. | Величина | Ед. | Источник |
|---|---|---|---|
| $RMSE$ | среднеквадратичная ошибка | — | выход |
| $y_i^{obs}$ | наблюдение | — | данные |
| $y_i^{sim}$ | модель | — | расчёт |

**Метод:** Root Mean Square Error

**Применимость:** calibrateParameters и валидация twin.

**Допущения:** Одинаковая длина рядов; Конечные значения

**Код:** `src/core/calibration/calibrator.ts`

export function layerResistance(thicknessM, lambdaWmK) {
  return thicknessM / lambdaWmK;                 // R = d / λ
}
export function assemblyResistance(layers, Rsi = 0, Rse = 0) {
  return Rsi + layers.reduce((s, l) => s + layerResistance(l.thicknessM, l.lambdaWmK), 0) + Rse;
}
export function uValue(resistance) {
  return 1 / resistance;                          // U = 1 / R_o
}

### Теплопотери: трансмиссия, вентиляция, инфильтрация

`src/core/thermal/formulas.ts`

```ts
export function transmissionLoss(U, areaM2, deltaT) {
  return U * areaM2 * deltaT;                      // Q = U·A·ΔT
}
export function airflowFromACH(ach, volumeM3) {
  return (ach * volumeM3) / 3600;                  // L = n·V / 3600, м³/с
}
export function ventilationLoss(flowM3s, rho, cp, deltaT) {
  return rho * cp * flowM3s * Math.max(0, deltaT); // Q = ρ·c_p·L·ΔT
}
export function infiltrationLoss(ach, volumeM3, rho, cp, deltaT) {
  return ventilationLoss(airflowFromACH(ach, volumeM3), rho, cp, deltaT);
}
```

### Удельная теплозащитная характеристика k_ob и годовая энергия (СП 50)

`src/core/thermal/sp50/calculations.ts`

```ts
export function calculateKob({ fragments, heatedVolumeM3 }) {
  // k_ob = (Σ n_t·A_i / R_pr,i) / V_от
  return fragments.reduce((s, f) => s + (f.nt * f.areaM2) / f.reducedResistance_m2K_W, 0) / heatedVolumeM3;
}
export function calculateHeatingEnergyCharacteristic(i) {
  // q_от = k_ob + k_vent − β·(q_быт + q_сол)
  return i.kob_W_m3K + i.ventilationCharacteristic_W_m3K
    - i.betaGainUseFactor * (i.internalGainCharacteristic_W_m3K + i.solarGainCharacteristic_W_m3K);
}
export function calculateAnnualHeatingEnergy(gsop, V, q) {
  return 0.024 * gsop * V * q;                     // Q_год = 0.024·ГСОП·V·q_от, кВт·ч
}
```
